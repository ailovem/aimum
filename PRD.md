## 1. 项目概述

### 1.1 项目背景

**AImum = OpenClaw + Claude Cowork**

一人公司 AI 操作系统，让创业者像搭积木一样组建 AI 团队。

### 1.2 核心理念

```
一个人 = 一个 AI 团队
- 销售部（获客、跟进、转化）
- 市场部（内容、投放、品牌）
- 财务部（记账、报表、风控）
- 行政部（日程、提醒、协作）
- 技术部（开发、运维、部署）
```

### 1.3 项目目标

| 目标 | 指标 |
|------|------|
| MVP 上线 | 2 周内 |
| 注册用户 | 1000 人/周 |
| 付费转化 | 5% |
| 续费率 | 60% |

| 无令牌系统 | 缺计费能力 |

### 4.2 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 Web | React + Tailwind | 轻量、组件化 |
| 前端 PWA | Vite + Workbox | 离线支持 |
| 后端 | Node.js + Express | 简单高效 |
| 数据库 | PostgreSQL + Prisma | 关系型、强类型 |
| 缓存 | Redis | 性能提升 |
| 部署 | Vercel | 免费、好用 |
| 域名 | 阿里云 | 已注册 |
| AI | DeepSeek API | 便宜、好用 |
| 监控 | 飞书机器人 | 实时推送 |

### 4.3 插件架构

```
插件市场
├── 插件格式 (plugin.json)
│   ├── id: 唯一标识
│   ├── name: 名称
│   ├── version: 版本
│   ├── permissions: 权限列表
│   ├── actions: 动作列表
│   └── installUrl: 安装地址
│
├── 插件分类
│   ├── 销售部 (sales)
│   ├── 市场部 (marketing)
│   ├── 财务部 (finance)
│   ├── 行政部 (admin)
│   ├── 技术部 (tech)
│   └── 自定义 (custom)
│
└── 插件市场 API
    ├── GET /plugins (列表)
    ├── GET /plugins/:id (详情)
    └── POST /plugins/install (安装)
```
